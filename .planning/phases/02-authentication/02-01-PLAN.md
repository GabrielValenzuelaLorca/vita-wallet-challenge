  <action>
1. Add `gem "jwt", "~> 2.7"` to backend/Gemfile and run `bundle install`.

2. Create `backend/app/services/jwt_service.rb`:
   - Class with class methods `encode(user_id:)` and `decode(token:)`
   - `encode`: builds payload `{ user_id: user_id, exp: 24.hours.from_now.to_i }`, signs with `Rails.application.secret_key_base` using HS256 algorithm via `JWT.encode`
   - `decode`: calls `JWT.decode(token, secret, true, algorithm: 'HS256')`, returns the payload hash `{ "user_id" => ... }`
   - Rescues `JWT::DecodeError` and `JWT::ExpiredSignature`, returns `nil` on any failure
   - Private constant or method for the secret key (`Rails.application.secret_key_base`)

3. Create `backend/app/services/auth_service.rb`:
   - Class with class methods `register(email:, password:)` and `authenticate(email:, password:)`
   - `register`: creates `User.create!(email: email, password: password)`, then generates token via `JwtService.encode(user_id: user.id)`, returns `{ user: user, token: token }`
   - If user creation fails (validation), raises `ActiveRecord::RecordInvalid`
   - `authenticate`: finds user by `User.find_by(email: email)`, calls `user&.authenticate(password)`. If user is nil or authenticate returns false, returns `nil`. Otherwise generates token via `JwtService.encode` and returns `{ user: user, token: token }`

4. Create `backend/app/controllers/auth_controller.rb`:
   - Inherits from `ApplicationController`
   - `register` action: extracts `email` and `password` from `params`, calls `AuthService.register`, uses `UserSerializer` to serialize the user, returns `render_success(data: { token: result[:token], user: serialized_user }, status: :created)`. Rescues `ActiveRecord::RecordInvalid` and returns `render_error(code: "validation_error", message: e.record.errors.full_messages.join(", "), status: :unprocessable_entity)`
   - `login` action: extracts `email` and `password` from `params`, calls `AuthService.authenticate`. If nil, returns `render_error(code: "invalid_credentials", message: "Invalid email or password", status: :unauthorized)`. Otherwise returns `render_success(data: { token: result[:token], user: serialized_user })`
   - Private method `auth_params` that permits `:email, :password`

5. Modify `backend/app/controllers/application_controller.rb`:
   - Add `authenticate_user!` private method: reads `Authorization` header, extracts bearer token, decodes via `JwtService.decode`. If decode fails or user not found, returns `render_error(code: "unauthorized", message: "Invalid or expired token", status: :unauthorized)`. Sets `@current_user` on success.
   - Add `current_user` helper method returning `@current_user`
   - Keep existing `render_success` and `render_error` methods unchanged

6. Update `backend/config/routes.rb`:
   - Add `post "auth/register", to: "auth#register"` and `post "auth/login", to: "auth#login"` (flat routes under `/auth/`, not namespaced)
