**1. ExchangeService (`backend/app/services/exchange_service.rb`)**

PORO service object following the project's established class method pattern (like AuthService, PriceService). Single public class method:

```ruby
ExchangeService.execute(user:, source_currency:, target_currency:, source_amount:)
```

Returns a Hash with `{ success: bool, transaction: Transaction, error_code: String?, error_message: String? }`.

Implementation flow inside `ActiveRecord::Base.transaction`:

1. **Validate currencies**: Both must be in `Wallet::CURRENCIES`. Both must exist for the user. Source and target must differ. If invalid, return early with error (do NOT create a Transaction for invalid input -- return `{ success: false, transaction: nil, error_code: "invalid_currencies", error_message: "..." }`).

2. **Coerce source_amount**: Convert to BigDecimal via `BigDecimal(source_amount.to_s)`. Validate it is positive. If not, return error with code `"invalid_amount"`.

3. **Check balance**: Load `user.wallets.lock.find_by!(currency: source_currency)` (use `lock` for pessimistic locking within the transaction). Compare `source_wallet.balance >= source_amount_decimal`. If insufficient, create a `Transaction` with status `"rejected"`, `rejection_reason: "insufficient_balance"`, set `target_amount: 0` and `exchange_rate: 0`, then return `{ success: false, transaction: txn, error_code: "insufficient_balance" }`.

4. **Fetch price**: Call `PriceService.fetch_prices`. Determine the exchange rate based on the direction:
   - If source is fiat (USD/CLP) and target is crypto (BTC/USDC/USDT): Look up `prices[target_currency][source_currency]`. This gives "price of 1 crypto in source fiat". So `target_amount = source_amount / BigDecimal(price)`.
   - If source is crypto and target is fiat: Look up `prices[source_currency][target_currency]`. This gives "price of 1 crypto in target fiat". So `target_amount = source_amount * BigDecimal(price)`.
   - If both are fiat or both are crypto: Cross-rate through USD. For example fiat-to-fiat: convert source to USD first, then USD to target. For crypto-to-crypto: convert source crypto to USD, then USD to target crypto. Use the same price lookup patterns.
   - The `exchange_rate` persisted on the Transaction is `target_amount / source_amount` (the effective rate of the operation).
   - If `PriceClient::ApiError` is raised during price fetch, create a rejected Transaction with `rejection_reason: "price_fetch_failed"`, `target_amount: 0`, `exchange_rate: 0`.

5. **Execute exchange**: 
   - `source_wallet.balance -= source_amount_decimal` then `source_wallet.save!`
   - Load `target_wallet = user.wallets.lock.find_by!(currency: target_currency)`
   - `target_wallet.balance += target_amount` then `target_wallet.save!`
   - Create Transaction with status `"completed"`, all amounts and exchange_rate populated.

6. **Rollback on any unexpected error**: The `ActiveRecord::Base.transaction` block ensures rollback. Rescue `ActiveRecord::RecordInvalid` or `StandardError` inside the transaction block, and if a pending transaction was created, update it to `rejected` with `rejection_reason: "internal_error"`. Re-raise to trigger rollback.

CRITICAL RULES:
- ALL arithmetic MUST use BigDecimal. Never call `.to_f` on any monetary value.
- Use `BigDecimal(string)` to convert price strings from PriceService.
- The `lock` call on wallets is essential to prevent race conditions.
- Wrap the entire debit+credit+transaction-create in `ActiveRecord::Base.transaction`.

Helper method `fiat_currency?(currency)` returns true for USD/CLP, false for BTC/USDC/USDT.
