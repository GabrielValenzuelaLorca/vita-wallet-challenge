<task type="auto">
  <name>Task 1: Initialize Rails API-only app with PostgreSQL and create DB schema</name>
  <files>
    backend/ (entire new Rails app)
    backend/Gemfile
    backend/config/database.yml
    backend/config/application.rb
    backend/config/routes.rb
    backend/db/migrate/*_create_users.rb

    backend/db/migrate/*_create_transactions.rb
    backend/app/models/user.rb
    backend/app/models/wallet.rb
    backend/app/models/transaction.rb
  </files>
  <action>
    1. Generate a new Rails API-only app inside /backend:
       `rails new backend --api --database=postgresql --skip-javascript --skip-asset-pipeline --skip-action-mailer --skip-action-mailbox --skip-action-text --skip-active-storage --skip-action-cable -T`
       (use -T to skip minitest, we use RSpec in plan 01-03)

    2. Configure database.yml for local PostgreSQL (development and test databases: vita_wallet_development, vita_wallet_test).

    3. Add to Gemfile: `bcrypt` (for has_secure_password), `rack-cors` (for frontend CORS).

    4. Create migration for `users` table:
       - id: bigint (default Rails PK)
       - email: string, NOT NULL, unique index
       - password_digest: string, NOT NULL
       - timestamps

    5. Create migration for `wallets` table:
       - id: bigint
       - user_id: bigint, NOT NULL, foreign key to users
       - currency: string, NOT NULL (one of: USD, CLP, BTC, USDC, USDT)
       - balance: decimal, precision: 20, scale: 8, NOT NULL, default: 0
       - timestamps
       - Add unique compound index on [user_id, currency] to enforce one wallet per currency per user

    6. Create migration for `transactions` table:
       - id: bigint
       - user_id: bigint, NOT NULL, foreign key to users
       - source_currency: string, NOT NULL
       - target_currency: string, NOT NULL
       - source_amount: decimal, precision: 20, scale: 8, NOT NULL
       - target_amount: decimal, precision: 20, scale: 8, NOT NULL
       - exchange_rate: decimal, precision: 30, scale: 18, NOT NULL
       - status: string, NOT NULL, default: "pending" (values: pending, completed, rejected)
       - rejection_reason: string, nullable
       - timestamps
       - Index on [user_id, status] for filtered queries

    7. Create User model:
       - `has_secure_password`
       - `has_many :wallets, dependent: :destroy`
       - `has_many :transactions, dependent: :destroy`
       - Validations: email presence, uniqueness (case-insensitive), format; password length minimum 6

    8. Create Wallet model:
       - `belongs_to :user`
       - CURRENCIES constant: %w[USD CLP BTC USDC USDT].freeze
       - Validations: currency presence + inclusion in CURRENCIES, balance numericality >= 0
       - Scoped uniqueness of currency per user_id

    9. Create Transaction model:
       - `belongs_to :user`
       - STATUSES constant: %w[pending completed rejected].freeze
       - Validations: source_currency, target_currency, source_amount, target_amount, exchange_rate, status presence
       - status inclusion in STATUSES

    10. Run `bundle install`, `rails db:create`, `rails db:migrate`.

    11. Configure CORS in config/initializers/cors.rb to allow localhost:5173 (Vite dev server) with credentials.

    12. Set up base ApplicationController with a `render_success` and `render_error` helper for consistent API envelope:
        - Success: `{ data: ..., meta: ... }` with appropriate HTTP status
        - Error: `{ error: { code: "snake_case_code", message: "Human readable" } }` with appropriate HTTP status
  </action>
  <verify>
    cd backend && bundle exec rails db:migrate:status | grep -c "up" should show 3 migrations up.
    cd backend && bundle exec rails runner "puts User.table_name; puts Wallet.table_name; puts Transaction.table_name" should output users, wallets, transactions without error.
  </verify>
  <done>
    Rails API-only app exists in /backend. PostgreSQL databases created. 3 tables (users, wallets, transactions) migrated with correct column types and decimal precision. Models have associations and validations. ApplicationController has envelope response helpers. CORS configured for frontend dev server.

       Print summary after seeding: "Seeded N users with M wallets each".

    2. Create app/serializers/base_serializer.rb:
       A simple PORO serializer base class that provides:
       - `initialize(object, options = {})` storing @object and @options
       - `as_json` method that subclasses override to return a hash
       - `serialize` class method that wraps `as_json` output in `{ data: ... }`
       - `serialize_collection(collection, meta: {})` class method that wraps array in `{ data: [...], meta: ... }`
       This is intentionally lightweight (no jbuilder, no active_model_serializers) for full control over response shape.

    3. Create app/serializers/user_serializer.rb:
       - Serializes: id, email, created_at
       - Excludes: password_digest (NEVER expose)

    4. Create app/serializers/wallet_serializer.rb:
       - Serializes: id, currency, balance (as string to preserve decimal precision in JSON)
       - Note: balance.to_s("F") to get fixed-point string representation

    5. Run `rails db:seed` and verify output.
