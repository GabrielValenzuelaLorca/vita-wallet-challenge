  <action>
    **WalletsController** (`backend/app/controllers/wallets_controller.rb`):
    - `before_action :authenticate_user!`
    - Single action `index` that fetches `current_user.wallets` and returns via `WalletSerializer.serialize_collection`
    - Returns `{ data: [{id, currency, balance}, ...], meta: {} }`

    **PriceClient** (`backend/app/services/price_client.rb`):
    - Low-level HTTP client class that calls `https://api.stage.vitawallet.io/api/prices_quote`
    - Method `fetch_prices` returns a hash like `{ "BTC" => { "USD" => "67432.50", "CLP" => "..." }, "USDC" => {...}, "USDT" => {...} }` — the exact shape depends on the external API, but normalize it into `{ currency_code => { quote_currency => price_string } }`
    - Uses `Net::HTTP` with a configurable timeout (default 5 seconds via `ENV.fetch("PRICE_API_TIMEOUT", 5).to_i`)
    - Raises `PriceClient::ApiError` on timeout (`Net::OpenTimeout`, `Net::ReadTimeout`), HTTP non-2xx responses, or JSON parse failure
    - `PriceClient::ApiError` is a custom error class defined inside the file with a `code` attribute (`:timeout`, `:server_error`, `:invalid_response`)

    **StubPriceClient** (`backend/app/services/stub_price_client.rb`):
    - Same interface as PriceClient (`fetch_prices` method)
    - Returns hardcoded realistic prices: `{ "BTC" => { "USD" => "67432.50", "CLP" => "62500000.00" }, "USDC" => { "USD" => "1.00", "CLP" => "925.00" }, "USDT" => { "USD" => "1.00", "CLP" => "925.00" } }`
    - No HTTP calls, no external dependencies — always succeeds
    - Used in development and test environments

    **PriceService** (`backend/app/services/price_service.rb`):
    - Class method `fetch_prices(client: nil)` — accepts optional client injection, defaults to the configured global client
    - Wraps fetch in `Rails.cache.fetch("crypto_prices", expires_in: 30.seconds)` block
    - Inside cache block, calls `client.fetch_prices` and returns the result
    - Rescues `PriceClient::ApiError` and re-raises as a service-level error with descriptive message
    - The cache key and TTL should be constants: `CACHE_KEY = "crypto_prices"`, `CACHE_TTL = 30.seconds`

    **PricesController** (`backend/app/controllers/prices_controller.rb`):
    - `before_action :authenticate_user!`
    - Single action `index` that calls `PriceService.fetch_prices` and wraps result in `render_success(data: prices)`
    - Rescues `PriceClient::ApiError` and returns `render_error(code: "price_service_unavailable", message: error.message, status: :service_unavailable)` (503)

    **Routes** — Add to existing `config/routes.rb`:
    - `get "balances", to: "wallets#index"`
    - `get "prices", to: "prices#index"`

    **Initializer** (`backend/config/initializers/price_client.rb`):
    - Sets `Rails.application.config.price_client` to `StubPriceClient.new` in development/test, `PriceClient.new` in production
    - PriceService reads from `Rails.application.config.price_client` as default when no client is injected

    Follow established patterns: thin controller delegates to service, PORO serializers, API envelope `{ data, meta }` for success, `{ error: { code, message } }` for errors.
